## 2. 게임 완성도 및 연산 라이프사이클 
* **상세 구현:** 터널 내부에 무작위 아핀 변환(Affine Transform) 회전 및 위치 셔플이 적용된 기둥 장애물 10개를 실시간으로 배치했습니다. 매 프레임 비행기와 장애물의 중심점 간 3D 유클리드 거리를 계산하여 충돌을 감지하는 **Collision Detection** 알고리즘을 수행합니다. 장애물 충돌 감지 시 그래픽스 파이프라인의 ClearColor를 세련된 적색(`0xff2222`)으로 순간 반전시켜 시각적 충격 피드백을 주며, 0.8초 후 전체 상태 행렬과 오브젝트 위치를 초기화하는 게임 재시작 라이프사이클을 견고하게 구축했습니다.

## 3. 글로벌 일루미네이션(GI) 기술 적용 상세 설명
* **강의 이론 매핑:** 난반사(Diffuse) 성질의 간접광은 저주파(Low-frequency) directional 신호라는 특성을 가집니다. 본 구현에서는 무거운 레이 바운스 적분 연산 대신 **L1 단계의 구면 조화 함수(Spherical Harmonics, SH)** 기저 함수 개념을 프레임워크 셰이더 연산으로 근사화했습니다.
* **상세 구현:** 커스텀 `ShaderMaterial`을 설계하여 실시간으로 위치와 색상이 변하는 구체 조명의 유니폼 변수(`uNeonPos`, `uNeonColor`)를 주입받습니다. 프래그먼트 셰이더 내에서 표면 법선 벡터($N$)와 광원 방향 벡터($L$)의 내적 연산인 $abs(N \cdot L)$ 및 거리 감쇄식을 결합하여, 조명이 터널 내부 전체에 은은하고 쨍하게 튕겨 번지는 **간접광(Indirect Radiance) 전파 효과**를 고성능 실시간 GI 형태로 완벽히 시각화했습니다.

## 4. 구현 내용 연동 및 행렬 직접 제어 
* **쿼터니언 Slerp 회전 보간:** 오일러 각의 짐벌 락(Gimbal Lock)을 원천 차단하기 위해 사원수 체계를 도입했으며, `quaternion.slerp` 메서드를 사용하여 평형상태에서 좌우 조작 시 기체가 진행 방향으로 부드럽게 기우뚱(Roll 기동) 가속/감속 보간되도록 제어했습니다.
* **카메라 Position Lerp:** O키(1인칭 콕핏 뷰), P키(3인칭 백뷰) 입력 시 카메라 위치 행렬이 불연속적으로 순간 이동하지 않고, 타겟 좌표계까지 미끄러지듯 부드럽게 도달하도록 선형 보간(Lerp) 연산을 적용했습니다.
* **변환 행렬 직접 제어 (Matrix Compose):** Three.js의 자동 행렬 갱신인 `matrixAutoUpdate = false`로 차단하고, 연산된 스케일, 쿼터니언, 이동 벡터 데이터를 `.matrix.compose()` 메서드로 $4 \times 4$ 변환 행렬에 직접 수동 결합하여 그래픽스 파이프라인 최적화를 달성했습니다.
